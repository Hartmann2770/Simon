## Key Findings

1. Cognitive load theory (CLT) is grounded in an evolutionary account of human cognition: the brain has evolved dedicated machinery for acquiring *biologically primary* knowledge (e.g., language, face recognition) but not for *biologically secondary* knowledge (e.g., reading, mathematics), which requires explicit instruction.
2. Working memory is severely limited in both capacity (roughly four elements) and duration when processing novel information, while long-term memory is effectively unlimited; this asymmetry is the architectural basis for instructional design.
3. Three types of cognitive load are identified: **intrinsic load** (determined by element interactivity of the material itself), **extraneous load** (caused by poor instructional design), and **germane load** (effort devoted to schema formation and automation).
4. The fundamental instructional principle is to minimise extraneous load so that total load remains within working memory capacity — but when intrinsic load itself exceeds capacity, it must also be managed (e.g., by isolating interacting elements).
5. Worked examples are one of CLT's most powerful prescriptions: novices learn more from studying solved problems than from solving equivalent problems, because problem solving consumes working memory on the search process rather than on schema acquisition.
6. The *expertise reversal effect* complicates prescriptions: as learners gain knowledge, instructional supports that helped novices become redundant or even harmful.
7. Instructional procedures derived from CLT include the worked-example effect, split-attention effect, redundancy effect, modality effect, and element-interactivity effect.

## Methodology

Theoretical chapter synthesising CLT's development from 1988 to 2011. Reviews empirical studies supporting each CLT effect and situates the theory within evolutionary psychology. No new empirical data; this is a comprehensive theoretical statement of the mature theory.

## Relevance

The canonical reference for CLT as a unified theory. Essential for anyone designing instructional materials, because CLT directly translates cognitive architecture into design rules. Particularly important for understanding *why* certain multimedia and instructional design strategies work.
